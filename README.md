# Site Index Hub

## 项目简介

Site Index Hub 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常用于记录、展示或索引某些信息，不限定于特定网站或域名。本仓库旨在以简洁、清晰的方式整理和托管这些静态页面，方便访问与维护。

## 目录说明

```
site-index-hub/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 公共资源文件（如 CSS、JS、图片）
├── index.html      # 仓库首页，可跳转到各页面
└── README.md       # 本说明文件
```

- **pages/**：每个 HTML 页面为一个独立文件，命名尽量体现内容主题。
- **assets/**：存放页面共用的样式、脚本或图片资源，避免重复引用。
- **index.html**：可选的入口页面，用于快速导航至各归档页面。

## 页面归档说明

归档的 HTML 页面均为静态文件，不依赖后端服务。页面内容可能包含数据展示、链接集合、信息整理等，具体用途因页面而异。每个页面独立运行，互不干扰。

### 页面命名规范

- 使用小写字母、数字和连字符（`-`）命名，如 `example-page.html`。
- 文件名应简要反映页面内容，避免过长或含特殊字符。

### 添加新页面

1. 在 `pages/` 目录下创建新的 HTML 文件。
2. 如需引用公共资源，请使用相对路径指向 `../assets/`。
3. 可选：更新 `index.html` 以添加新页面的链接。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面会通过 Git 提交记录体现。
- 欢迎通过 Issue 提出页面归档建议或报告链接失效等问题。
- 不接受直接修改归档页面内容的 Pull Request（除非涉及明显错误），以确保页面稳定性。
